# Phase 05 · Track 02 — Security & Privacy

Protect sensitive data and maintain trust with schools and learners.

## A. Security Program
- [ ] Conduct comprehensive threat modeling for each service.
- [ ] Schedule external penetration tests and remediate findings.
- [ ] Implement vulnerability management process with SLA tiers.
- [ ] Enforce least-privilege IAM across infrastructure and operations.

## B. Data Protection
- [ ] Ensure encryption at rest (KMS-managed) and in transit (TLS 1.2+).
- [ ] Apply field-level encryption for highly sensitive attributes.
- [ ] Monitor data exfiltration attempts with DLP tooling.
- [ ] Automate secure deletion requests and backups sanitization.

## C. Privacy & Compliance
- [ ] Map data flows and maintain Records of Processing Activities (RoPA).
- [ ] Produce compliance evidence for FERPA/GDPR and partner audits.
- [ ] Implement privacy-by-design review in development lifecycle.
- [ ] Provide data subject access request (DSAR) tooling with SLAs.

## D. Security Culture
- [ ] Deliver regular security training tailored to roles.
- [ ] Run phishing simulations and track improvement metrics.
- [ ] Establish secure coding guidelines and review checklists.
- [ ] Maintain security champions program across squads.
